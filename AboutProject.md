# Analyzing E-Learning Platform Purchases Using MySQL – Project Summary

## Overview

This project analyzes purchase transactions from an e-learning platform using MySQL. The database stores information about learners, courses, and course purchases. 
The analysis focuses on understanding learner behavior, course popularity, revenue generation, and business performance through SQL queries.

## Database Structure

The project consists of three relational tables:

### 1. Learners

Stores learner information:

* Learner ID
* Full Name
* Country

### 2. Courses

Stores course details:

* Course ID
* Course Name
* Category
* Unit Price

### 3. Purchases

Stores transaction records:

* Purchase ID
* Learner ID
* Course ID
* Quantity Purchased
* Purchase Date

The tables are connected using foreign key relationships, ensuring data integrity and consistency.

## Key SQL Concepts Used

* CREATE DATABASE
* CREATE TABLE
* PRIMARY KEY
* FOREIGN KEY
* AUTO_INCREMENT
* INNER JOIN
* LEFT JOIN
* RIGHT JOIN
* GROUP BY
* ORDER BY
* HAVING
* Aggregate Functions (SUM, COUNT)
* Data Formatting using FORMAT()

## Analysis Performed

### 1. Purchase Transaction Analysis

Combined learner, course, and purchase information to display complete transaction details, including:

* Learner name
* Country
* Course purchased
* Course category
* Quantity purchased
* Total purchase amount
* Purchase date

### 2. Learner Spending Analysis

Calculated total spending by each learner to identify:

* Highest spending learners
* Revenue contribution by customer
* Customer purchasing behavior

### 3. Course Performance Analysis

Analyzed total quantity sold for each course to determine:

* Most popular courses
* Best-selling educational content
* Demand trends

### 4. Category Revenue Analysis

Measured revenue generated by each course category and counted unique learners in each category to understand:

* Most profitable categories
* Customer interest areas
* Market demand

### 5. Multi-Category Learner Analysis

Identified learners who purchased courses from multiple categories, helping to:

* Recognize highly engaged customers
* Understand cross-category learning behavior

### 6. Unsold Course Analysis

Found courses that have not been purchased by any learner, helping the business:

* Identify low-performing courses
* Improve marketing strategies
* Optimize course offerings

## Business Insights

1. Learners from different countries actively purchased online courses, demonstrating the platform's international reach.

2. Some learners generated significantly higher revenue than others, indicating valuable customer segments.

3. Programming, Analytics, AI, and Development courses showed strong market demand.

4. Revenue varied across categories, helping identify the most profitable learning domains.

5. Certain learners purchased courses from multiple categories, indicating strong engagement and potential for upselling.

6. The analysis of unsold courses helps identify content that may require promotion, updates, or removal.

## Project Objective

The objective of this project is to design a relational database for an e-learning platform and use SQL queries to analyze learner purchases, 
evaluate course performance, measure revenue generation, and generate business insights that support data-driven decision-making.

## Conclusion

This project successfully demonstrates how MySQL can be used to manage and analyze e-learning platform data. By leveraging relational database 
concepts and SQL analytical queries, valuable insights were obtained regarding learner spending, course popularity, category performance, and overall 
revenue generation. The findings can help the organization improve course offerings, enhance marketing strategies, increase customer engagement, and drive future business growth.
